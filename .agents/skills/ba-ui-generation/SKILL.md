# UI Prototype Generation

## Workflow

1. Read the data model and scenario analysis
2. Determine UI type needed:
   - Simple form / Multi-step form / Dynamic form
   - Query/list page / Dashboard
   - Interactive page / Landing page
3. Generate HTML prototype with appropriate components
4. Include form validation and interaction logic

## Input

- Data model, scenario analysis, function list

## Output

- HTML UI prototypes

## Source Reference

See `ba-copilot/03.UI生成/` directory (3.1 through 3.8) for templates per UI type.
