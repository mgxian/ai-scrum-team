# Requirement Optimization

## Context

After PO fills in the business background questionnaire, this skill
combines the original requirement with the answers to produce an optimized,
structured requirement document that serves as the foundation for all
subsequent analysis.

## Workflow

1. Read the original requirement and PO's questionnaire answers
2. Identify gaps, ambiguities, and implicit assumptions
3. Generate an optimized requirement document with clear scope and boundaries
4. Submit for PO confirmation (blocking — all subsequent work depends on this)

## Input

- Original requirement
- Completed business background questionnaire

## Output

- Optimized requirement document
