# Change Impact Analysis

## Workflow

1. Read the change request and current system documentation
2. Analyze impact on:
   - Data model (table/field changes)
   - UI prototypes (affected pages)
   - External dependencies (API changes)
   - Settlement/billing logic (if applicable)
3. Estimate change scope and risk
4. Output impact analysis report

## Input

- Change request, current system documentation

## Output

- Impact analysis report per dimension
