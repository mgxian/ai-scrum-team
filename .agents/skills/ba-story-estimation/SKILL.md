# Requirement Estimation & Story Mapping

## Workflow

1. Read the User Story list with dependencies
2. Estimate effort per story
3. Build user story map (backbone → walking skeleton → iterations)
4. Schedule stories into Sprints based on dependencies and capacity
5. Output story map and Sprint schedule

## Input

- User Story list, dependency analysis, team capacity

## Output

- User story map, Sprint schedule
