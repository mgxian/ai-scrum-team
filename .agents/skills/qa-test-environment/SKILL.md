# Test Environment & Data Management

## Workflow

1. Read the architecture design and deployment topology
2. Define environment requirements (dev, test, staging, demo)
3. Plan test data seeding strategy
4. Define data isolation and cleanup procedures
5. Output environment management plan

## Input

- Architecture design, deployment topology

## Output

- Test environment and data management plan
