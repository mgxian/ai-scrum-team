# Sprint Retrospective Analysis

## Workflow

1. Read Sprint execution data (completion, reviews, tests, deployments)
2. Analyze metrics:
   - Completion rate (planned vs completed points)
   - Quality (review pass rate, test pass rate, security issues)
   - Efficiency (lead time, review time, blocked hours)
3. Generate improvement items categorized by target role
4. Identify recurring issues across Sprints
5. Output retrospective report

## Input

- Sprint execution data, historical Sprint reports

## Output

- Retrospective report with metrics and improvement items
