# Requirement Quality Scan

## Workflow

1. Read all BA analysis outputs
2. Check completeness (missing scenarios, roles, validations)
3. Check consistency (contradictions between documents)
4. Check testability (vague acceptance criteria)
5. Check ambiguity (unclear terms, missing definitions)
6. Output quality report with severity ratings

## Input

- All BA analysis outputs

## Output

- Quality scan report with issues and recommendations
