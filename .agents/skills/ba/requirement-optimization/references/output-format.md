# 输出格式

## 固定章节（所有需求必须输出）

### 一、业务概述

若类型A（内部平台/工具类）：
- 平台定位（模块在整体系统中的位置、职责边界）
- 服务对象（使用该模块的内部团队、角色及核心诉求）
- 协作关系（上下游团队职责分工、数据流转与协作触点）

若类型B（面向市场的业务）：
- 行业与业务类型
- 企业角色定位
- 核心商业模式

### 二、业务目标与诉求

- 当前痛点（按优先级排序）
- 期望达成的目标（尽量量化）
- 成本/效益考量（问卷未涉及则标注"待补充"）

### 三、约束条件

- 合规/政策要求
- 技术/工具限制
- 组织/协作限制

### 四、完整需求描述

写作要求：
- 以业务视角描述"要做什么"和"达到什么效果"
- 使用"系统应..."或"用户能够..."句式
- 每条需求可验证（能明确判断是否实现）
- 不拆解为功能点编号（如 FR-x.x）
- 不涉及实现方式、技术选型或系统架构

---

## 扩展章节（仅中大型需求输出，小功能跳过）

### 五、核心资源与能力

- 关键资源/资产
- 资源管理方式
- 核心团队结构

### 六、核心业务流程

- 高频日常场景
- 非标/特殊场景
- 现有核心流程
