# Stakeholder Analysis

## Workflow

1. Read the optimized requirement document
2. For each potential stakeholder category, assess relevance to this requirement
3. Identify specific concerns per stakeholder
4. Output as structured markdown table
5. Skip stakeholders with no direct relevance

## Stakeholder Categories

- Investors, Management, Compliance, Consultants
- Middle management, Requirement transmitters
- End users, Users' service targets
- Collaborating departments, Third-party companies

## Input

- Optimized requirement document

## Output

- Stakeholder concern matrix (markdown table)
