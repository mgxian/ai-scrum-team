# 生成角色权限矩阵（RBAC）。

要求：

1.根据以下业务需求：

【$ optimized_requirements

】

以下角色列表：

【$role_list】

以下功能列表

【$ function _ list】

生成角色权限矩阵，每个角色都要单独拆分一个表格，不要都整理在一个表格内。

2.为每个角色分配对应功能的访问权限（√表示有权限，空或×表示无权限）。

3.以清晰表格形式输出，包含以下列：

-
模块

-
功能

-
地址

- [
角色1]
- [
角色2]
- [
角色3] -（依此类推）

4.输出格式请使用Markdown表格

以下是矩阵示例：

|模块|功能|地址|授权|
|:--|:--|:--|:--|
|模块1 |功能1 | contract/sign/(id) | √ |
| |功能2 | contract/unsign/(id) | √ |
| |功能3 | contract/continue/(id) | √ |
| |功能4 | contract/list | √ |
| |功能5 | contract/detail/(id) | √ |
|模块2 |功能1 | settings/auto-withdraw | √ |
| |功能2 | withdraw/list | √ |
| |功能3 | withdraw/manual | √ |
