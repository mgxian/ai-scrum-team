## **动态表单核心概念定义**```动态表单=基础字段
+ N×动态条目组+提交逻辑

动态条目组=可重复添加/删除的字段集合

```

## **架构设计原则**1. **数据驱动**：表单结构完全由数据模型定义

2. **声明式配置**：通过配置描述表单行为，而非硬编码

3. **关注点分离**：

-
表单结构定义

-
业务逻辑处理

- UI
渲染表现

## **核心配置模型**```javascript //动态表单配置结构

//静态字段部分（始终存在）

staticSections: [

id: 'basic_info', title: '基本信息', fields: [ type: 'text', name: 'title', label: '标题', required: true , type: 'select', name: 'category', label: '分类', options: [...] ]

],

//动态条目部分（可重复添加）

dynamicSections: [

id: 'items', //对应数据字段名

title: '明细项目', minCount: 1, //最少条目数

maxCount: 50, //最多条目数

defaultItem: //新条目的默认值

name: '', quantity: 1, price: 0 , //每个条目的字段定义

fields: [

type: 'text', name: 'name', label: '项目名称', width: '40%', rules: [ required: true, message: '请输入名称' \] ,

type: 'number', name: 'quantity', label: '数量', width: '20%', min: 1, onChange: 'calculateTotal' //字段变化时触发的函数

], //条目标题模板（可选）

itemTitle: '项目
# index: name', //使用占位符动态显示

//联动规则

dependencies: [

sourceField: 'category', targetField: 'type', mapping: 'A': 'type1', 'B': 'type2'

]

]

```

## **数据流处理**### **数据结构**```javascript //表单数据示例

//静态字段

title: '采购申请单', category: '办公用品',

//动态字段（数组形式）

items: [ id: 'temp_1', name: '笔记本', quantity: 10, price: 5 , id: 'temp_2', name: '签字笔', quantity: 20, price: 2 ]

```

### **CRUD
操作规范**1. **添加条目**```javascript //生成唯一标识（前端临时ID）

const newItem = ...config.dynamicSections[0].defaultItem, id: `temp_$Date.now()\_$Math.random()\`

```

2. **删除条目** -至少保留`minCount`个条目

-
支持批量删除

3. **条目排序** -支持拖拽排序（如需要）

-
支持上移/下移操作

4. **数据验证** -单个条目内字段验证

-
跨条目业务规则验证

-
整体数量/金额限制验证

## **业务逻辑扩展点**### **1.
字段级逻辑**```javascript //字段配置扩展

type: 'computed', //计算字段

formula: 'quantity * price', readonly: true

type: 'conditional', //条件显示字段

showWhen: field: 'type', value: 'special'

```

### **2.
条目级逻辑**```javascript //条目验证规则

validateRules: //单个条目内验证

itemLevel: [ rule: 'price > cost', message: '售价必须大于成本' ], //跨条目验证

crossItem: [

rule: 'sum(quantity) <= 100', message: '总数量不能超过100'

]

```

### **3.
批量操作**```javascript //批量操作配置

batchOperations: [

name: 'copyFromPrevious', label: '复制上一行', icon: 'copy' ,

name: 'fillSameValue', label: '批量填充', field: 'unit', value: '个'

]
```

## **渲染层适配指南**### **基础组件要求**1. **动态容器组件** -渲染整个动态列表

-
管理条目的添加/删除/排序

2. **条目包装组件** -包裹单个条目的所有字段

-
提供删除/移动按钮

-
显示条目标题（如需要）

3. **字段渲染器** -根据字段类型选择对应UI组件

-
处理字段间的联动逻辑

### **状态管理要点**```javascript //推荐的状态结构

formData: , //完整表单数据

dynamicItems: //动态条目状态

'section1': items: [], //条目数据

errors: [], //验证错误

touched: [] //已交互标记

, metaData: //元数据

lastAddedId: null, focusItemId: null //当前焦点条目（用于滚动定位）

```

## **性能优化建议**### **针对大量条目的优化**1. **虚拟滚动**：条目超过50个时建议启用

2. **懒加载字段**：复杂字段在需要时再渲染

3. **防抖提交**：自动保存时使用防抖

4. **增量更新**：只更新变化的条目

### **内存管理**```javascript //清理未提交的临时数据

const cleanupTemporaryItems = (items) => return items.filter(item => !item.id?.startsWith('temp_')); ;
```

## **API
接口设计参考**```typescript //获取表单配置

GET /api/form-configs/:formType

//提交表单数据

POST /api/forms/:formType Body: staticFields: ... , dynamicSections: section1: [...], section2: [...]

//验证动态条目

POST /api/forms/:formType/validate-item Body: sectionId: 'items', itemData: ...
```

## **错误处理规范**1. **字段级错误**：在字段下方显示

2. **条目级错误**：在条目顶部显示

3. **跨条目错误**：在动态区域顶部显示

4. **提交时错误**：全局提示+定位到第一个错误

## **可访问性要求** 1.每个条目有唯一的`aria-labelledby` 2.添加/删除按钮有明确的ARIA标签

3.支持键盘导航：Tab键在条目内和条目间切换

4.屏幕阅读器能正确读取动态变化
