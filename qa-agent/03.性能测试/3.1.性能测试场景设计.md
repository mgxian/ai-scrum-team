### 一、性能测试场景识别

#### 1.1 场景选取原则

| 选取维度 | 说明 |
|:--|:--|
| 高频操作 | 用户日常使用频率最高的操作（如查询、列表加载） |
| 高并发场景 | 可能出现突发并发的场景（如抢购、批量审批、上班打卡） |
| 大数据量场景 | 涉及大量数据处理的场景（如报表导出、批量导入、数据迁移） |
| 长链路场景 | 调用链路长、涉及多个服务的场景（如下单全流程） |
| 资源密集场景 | CPU/内存/IO 密集型操作（如文件处理、复杂计算） |

| 场景编号 | 场景名称 | 场景类型 | 涉及 API | 并发用户数 | 持续时间 | 目标指标 |
|:--|:--|:--|:--|:--|:--|:--|

### 二、测试类型

| 测试类型 | 目的 | 执行方式 |
|:--|:--|:--|
| 基准测试 | 单用户下获取系统基准性能数据 | 单线程顺序执行所有场景 |
| 负载测试 | 验证预期负载下系统是否满足性能指标 | 逐步加压到目标并发数，持续 30 分钟 |
| 压力测试 | 找到系统性能拐点和极限 | 持续加压直到错误率 > 1% 或响应时间 > SLA 的 3 倍 |
| 稳定性测试 | 验证长时间运行下系统是否稳定 | 目标并发的 70% 负载，持续 2-4 小时 |
| 峰值测试 | 验证突发流量下系统的表现和恢复能力 | 瞬间加压到目标并发的 2 倍，观察恢复时间 |

### 三、性能指标基线

| 指标 | 说明 | 达标标准 | 告警阈值 |
|:--|:--|:--|:--|
| 平均响应时间 | 所有请求的平均耗时 | ≤ 200ms | > 500ms |
| P95 响应时间 | 95% 请求的耗时上限 | ≤ 500ms | > 1000ms |
| P99 响应时间 | 99% 请求的耗时上限 | ≤ 1000ms | > 2000ms |
| TPS | 每秒处理事务数 | 满足业务峰值需求 | < 预期的 80% |
| 错误率 | 请求失败比例 | ≤ 0.1% | > 0.5% |
| CPU 使用率 | 服务器 CPU 占用 | ≤ 70% | > 85% |
| 内存使用率 | 服务器内存占用 | ≤ 75% | > 85% |
| GC 频率 | 垃圾回收频率（Java 等） | Full GC ≤ 1次/小时 | Full GC > 3次/小时 |

### 四、测试数据准备

| 数据类型 | 数据量要求 | 准备方式 | 说明 |
|:--|:--|:--|:--|
| 基础数据 | 模拟生产数据量级 | 数据生成脚本 | 用户、组织、配置等基础数据 |
| 业务数据 | 生产数据量的 1-2 倍 | 数据生成脚本 + 脱敏生产数据 | 订单、工单等业务数据 |
| 历史数据 | 模拟 1 年以上的历史积累 | 批量生成 | 验证数据量增长对性能的影响 |
