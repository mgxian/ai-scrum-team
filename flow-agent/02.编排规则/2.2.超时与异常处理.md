## 超时与异常处理

定义 Flow Agent 在各环节遇到超时、Agent 执行失败、外部依赖不可用等异常时的处理策略。

---

### 一、Agent 执行超时

| Agent | 正常执行时限 | 超时处理 |
|:--|:--|:--|
| Code Agent | 按 Story 点数：1点=4h, 2点=8h, 3点=1.5d, 5点=2.5d, 8点=4d | 超时 → SM Agent 预警 → 再超 50% → PO 介入 |
| Review Agent | 2 小时 | 超时 → 自动重新触发 → 再超时 → SM Agent 预警 |
| QA Agent | 按 Story 点数：1点=2h, 2点=4h, 3点=1d, 5点=1.5d, 8点=2d | 超时 → SM Agent 预警 → 再超 50% → PO 介入 |
| Doc Agent | 2 小时 | 超时 → 自动重新触发 → 再超时 → 跳过文档，标记待补 |
| DevOps Agent | 30 分钟 | 超时 → 自动重试 1 次 → 再超时 → SM Agent 预警 |

### 二、Agent 执行失败

| 失败类型 | 处理策略 |
|:--|:--|
| Agent 内部错误（崩溃/异常退出） | 自动重启 Agent，重新执行当前任务，最多重试 2 次 |
| 输入数据格式错误 | 回退到上游 Agent，要求重新输出，附带格式校验错误信息 |
| 输出结果不符合预期 schema | 要求当前 Agent 重新执行，附带 schema 校验错误 |
| 连续 3 次失败 | 标记为异常，SM Agent 预警，PO 介入决策 |

### 三、外部依赖异常

| 异常类型 | 处理策略 |
|:--|:--|
| 外部 API 超时 | 重试 3 次（间隔 30s/60s/120s 指数退避）→ 降级处理或跳过 |
| 外部服务不可用 | 暂停依赖该服务的 Story，其他 Story 继续 → SM Agent 预警 |
| 数据库连接失败 | 重试 3 次 → 通知 DevOps Agent 排查 → 暂停所有执行中任务 |
| 构建/部署环境不可用 | 通知 DevOps Agent → 暂停部署队列 → 其他阶段不受影响 |

### 四、数据传递异常

| 异常类型 | 处理策略 |
|:--|:--|
| 上游产出为空 | 阻断流转，回退到上游 Agent 重新执行 |
| 上游产出格式不匹配 | 阻断流转，记录格式差异，通知上游 Agent 修正 |
| 上游产出内容不完整 | 下游 Agent 标记缺失项，回退到上游补充 |

### 五、异常恢复优先级

当多个异常同时发生时，按以下优先级处理：

1. 数据库/基础设施级异常（最高，影响全局）
2. 外部依赖异常（影响部分 Story）
3. Agent 执行失败（影响单个任务）
4. 超时预警（最低，可等待）

### 六、异常日志记录

所有异常必须记录以下信息：

```
异常编号：EX-{sprint}-{seq}
发生时间：{timestamp}
异常类型：{type}
影响 Story：{story_id}
影响 Agent：{agent_name}
当前状态：{current_state}
异常描述：{description}
处理策略：{action_taken}
处理结果：{result}
恢复时间：{recovery_time}
```
