## 测试环境与数据管理

核心指令：定义 QA Agent 执行测试所需的环境配置和测试数据准备规范。

---

### 一、测试环境分层

| 环境 | 用途 | 数据来源 | 外部依赖 | 管理者 |
|:--|:--|:--|:--|:--|
| 单元测试环境 | Code Agent 执行单元测试 | 内存数据库 + Mock | 全部 Mock | Code Agent |
| 集成测试环境 | QA Agent 执行集成测试 | 独立测试数据库 | Mock/Stub | DevOps Agent |
| E2E 测试环境 | QA Agent 执行端到端测试 | 接近生产的测试数据 | 沙箱环境 | DevOps Agent |
| 性能测试环境 | QA Agent 执行性能测试 | 生产级数据量 | 独立部署 | DevOps Agent |
| 预发布环境 | 最终验证 | 生产数据脱敏副本 | 真实外部系统（沙箱） | DevOps Agent |

### 二、测试数据准备规范

#### 2.1 数据准备原则

| 原则 | 说明 |
|:--|:--|
| 隔离性 | 每个 Story 的测试数据互不干扰，测试前初始化、测试后清理 |
| 可重复性 | 测试数据通过脚本生成，确保每次执行结果一致 |
| 完整性 | 覆盖正常数据、边界数据、异常数据、空数据 |
| 安全性 | 禁止使用真实个人信息，必须脱敏或使用虚构数据 |

#### 2.2 测试数据分类

| 数据类别 | 说明 | 示例 |
|:--|:--|:--|
| 正向数据 | 符合所有校验规则的合法数据 | 正常格式的手机号、合法金额范围 |
| 边界数据 | 处于校验边界的数据 | 最大长度字符串、最小金额、零值 |
| 异常数据 | 不符合校验规则的非法数据 | 格式错误、超出范围、特殊字符、SQL 注入串 |
| 状态数据 | 处于各种业务状态的预置数据 | 待审批的订单、已完成的工单、已过期的优惠券 |
| 权限数据 | 不同角色的账号和对应权限配置 | 管理员、普通用户、只读用户、无权限用户 |
| 并发数据 | 用于并发测试的共享资源数据 | 库存为 1 的商品、同一时段的会议室 |

#### 2.3 测试数据生命周期

```
测试开始前
  ↓
数据初始化（执行 setup 脚本，创建测试数据）
  ↓
测试执行（使用测试数据）
  ↓
测试结束后
  ↓
数据清理（执行 teardown 脚本，恢复环境）
```

### 三、环境就绪检查

QA Agent 在执行测试前，必须完成环境就绪检查：

| 检查项 | 检查方式 | 不通过处理 |
|:--|:--|:--|
| 数据库连接 | 执行连接测试 + 简单查询 | 通知 DevOps Agent |
| 外部服务可用性 | 调用各外部服务的健康检查接口 | Mock 降级或暂停测试 |
| 测试数据完整性 | 校验预置数据是否存在且状态正确 | 重新执行 setup 脚本 |
| 部署版本一致性 | 确认部署的代码版本与待测 Story 一致 | 通知 DevOps Agent 重新部署 |
| 配置正确性 | 校验测试环境配置（数据库、缓存、队列等） | 通知 DevOps Agent 修正 |

### 四、环境问题上报

当测试因环境问题失败时，QA Agent 必须区分"环境问题"和"代码问题"：

| 判定依据 | 归类 | 处理方式 |
|:--|:--|:--|
| 同一用例在本地通过、测试环境失败 | 环境问题 | 上报 DevOps Agent，不计入回退 |
| 外部服务超时/不可用导致失败 | 环境问题 | 上报 DevOps Agent，不计入回退 |
| 数据库连接池耗尽 | 环境问题 | 上报 DevOps Agent |
| 用例逻辑断言失败 | 代码问题 | 生成回退工单，回退到 Code Agent |
| 接口返回非预期错误码 | 代码问题 | 生成回退工单 |
