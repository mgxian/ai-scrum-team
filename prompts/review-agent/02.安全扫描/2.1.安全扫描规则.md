## 安全扫描规则

核心指令：根据以下代码变更内容，进行安全漏洞扫描，输出安全审查报告。

代码变更如下：

【$code_diff】

项目技术栈如下：

【$tech_stack】

---

### 一、注入类漏洞

| 检查项 | 规则 | 严重程度 |
|:--|:--|:--|
| SQL 注入 | 所有数据库查询必须使用参数化查询/预编译语句，禁止字符串拼接 SQL | 高 |
| XSS | 所有用户输入在输出到页面前必须转义，使用框架自带的转义机制 | 高 |
| 命令注入 | 禁止将用户输入直接拼接到系统命令中，使用白名单校验 | 高 |
| LDAP 注入 | LDAP 查询参数必须转义特殊字符 | 高 |
| 路径遍历 | 文件操作路径必须校验，禁止包含 `../`，使用白名单目录 | 高 |

### 二、认证与授权

| 检查项 | 规则 | 严重程度 |
|:--|:--|:--|
| 硬编码凭证 | 禁止在代码中硬编码密码、密钥、Token | 高 |
| 权限校验 | 每个 API 端点必须有权限校验，禁止仅依赖前端控制 | 高 |
| 越权访问 | 数据操作必须校验当前用户对目标资源的所有权/访问权 | 高 |
| 会话管理 | Token 必须设置合理过期时间，支持主动失效 | 中 |
| 密码存储 | 密码必须使用 bcrypt/scrypt/argon2 哈希存储，禁止 MD5/SHA1 | 高 |

### 三、数据安全

| 检查项 | 规则 | 严重程度 |
|:--|:--|:--|
| 敏感数据日志 | 禁止在日志中输出密码、身份证号、银行卡号等敏感信息 | 高 |
| 敏感数据传输 | 敏感数据必须通过 HTTPS 传输，响应中敏感字段需脱敏 | 高 |
| 敏感数据存储 | 敏感字段需加密存储（AES-256 或以上） | 高 |
| 文件上传 | 必须校验文件类型（MIME + 扩展名）、大小限制、存储路径隔离 | 中 |

### 四、依赖安全

| 检查项 | 规则 | 严重程度 |
|:--|:--|:--|
| 已知漏洞 | 第三方依赖不得包含已知高危 CVE 漏洞 | 高 |
| 版本锁定 | 依赖版本必须锁定（lock 文件），禁止使用 `latest` 或宽泛版本范围 | 中 |
| 许可证合规 | 引入新依赖需检查许可证兼容性（GPL 传染性等） | 中 |

### 五、API 安全

| 检查项 | 规则 | 严重程度 |
|:--|:--|:--|
| 限流 | 公开 API 必须配置限流策略 | 中 |
| CORS | CORS 配置不得使用 `*`，必须指定允许的域名列表 | 中 |
| 请求校验 | 所有入参必须做类型、长度、范围校验，拒绝非预期输入 | 高 |
| 防重放 | 关键操作（支付、审批）需防重放机制（幂等键/Nonce） | 高 |
