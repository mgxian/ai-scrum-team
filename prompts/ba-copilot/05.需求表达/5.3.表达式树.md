## 表达式树生成

根据业务流程如下：

【$flow】

生成的达式树。

表达式树是一种用于表示逻辑表达式组合的树形结构，其中：

-
每个终端节点表示一个具体的逻辑表达式，如"通过审批"或"结束时间
>开始时间" -终端节点不能是变量

-
表达式树通过逻辑节点表达AND、OR、XOR、NOT等关系的组合

## 输出格式要求

表达式树必须严格按照PlantUML格式输出，具体要求如下：

1.输出应为纯PlantUML代码，以`@startuml`开始，以`@enduml`结束

2.使用类图语法表示节点和连接

3.不包含任何额外文本、解释或注释

4.确保PlantUML代码完整且可执行

## PlantUML
表示规范

### 节点表示

-
每个节点用一个类表示

-
对于操作符节点，类名直接使用操作符：AND、OR、XOR或NOT -对于终端节点，类名是逻辑表达式字符串

-
如果终端节点包含特殊字符（如
>、<、=等）或空格，需要用双引号括起来

### 连接表示

-
节点之间的连接通过关联关系表示

-
使用箭头从父节点指向子节点

-
一个父节点最多可以有两个子节点

-
在关联线上添加标签来指定子节点角色：

- `leftChild`
：表示左子节点

- `rightChild`
：表示右子节点

-
对于NOT操作，只有一个子节点，可以使用`child`标签

### 结构要求

-
父节点的逻辑运算通过其名称表示

- AND
和OR是二元操作，需要两个子节点

- XOR
是二元操作，需要两个子节点

- NOT
是一元操作，只需要一个子节点

-
所有操作符都作为节点处理

## 示例

对于表达式树"((通过审批AND金额
>1000) OR紧急情况) AND NOT黑名单客户"，对应的PlantUML代码如下：

``` @startuml class AND class OR class "通过审批" class "金额
>1000" class "紧急情况" class NOT class "黑名单客户"

AND --> OR : leftChild AND --> NOT : rightChild OR --> "通过审批" : leftChild OR --> "金额
>1000" : rightChild NOT --> "黑名单客户" : child @enduml
```
