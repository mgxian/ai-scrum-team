## 技术选型分析

核心指令：根据以下输入信息，对系统的技术选型进行全面分析，输出技术选型决策文档。

结构化需求文档如下：

【$requirements_doc】

功能列表如下：

【$function_list】

非功能性需求如下：

【$non_functional_requirements】

现有技术栈如下（如有）：

【$existing_tech_stack】

---

### 一、技术选型维度

请从以下维度逐一分析并给出推荐方案：

1. **应用架构风格**
   - 单体 / 微服务 / Serverless / 事件驱动 / 混合架构
   - 选择依据：团队规模、业务复杂度、部署频率、扩展需求

2. **前端技术栈**
   - 框架选择（React / Vue / Angular 等）
   - 状态管理方案
   - UI 组件库
   - 构建工具

3. **后端技术栈**
   - 语言与框架
   - API 风格（RESTful / GraphQL / gRPC）
   - 认证授权方案

4. **数据存储**
   - 关系型数据库 / NoSQL / 混合
   - 缓存方案
   - 文件存储方案
   - 搜索引擎（如需要）

5. **消息与集成**
   - 消息队列选型
   - 事件总线方案
   - 第三方集成方式

6. **基础设施**
   - 容器化 / 编排方案
   - CI/CD 工具链
   - 监控与日志方案

### 四、技术选型决策记录（ADR）

对每个关键决策，按以下格式记录：

- **决策编号**：ADR-001
- **决策标题**：
- **背景**：为什么需要做这个决策
- **决策**：最终选择了什么
- **理由**：为什么选择它
- **后果**：这个决策带来的正面和负面影响
- **状态**：已采纳 / 待讨论 / 已废弃
