### 一、API 设计规范

1. **URL 规范**
   - 资源命名使用复数名词：`/api/v1/orders`
   - 层级关系用路径表达：`/api/v1/orders/{orderId}/items`
   - 操作类用动词：`/api/v1/orders/{orderId}/actions/cancel`

2. **HTTP 方法语义**
   - GET：查询（幂等）
   - POST：创建
   - PUT：全量更新（幂等）
   - PATCH：部分更新
   - DELETE：删除（幂等）

3. **响应规范**
   - 统一响应结构：`{ code, message, data, timestamp }`
   - 分页结构：`{ list, total, pageNum, pageSize }`
   - 错误码体系：模块编码 + 错误类型 + 序号

4. **版本管理**：URL 路径版本（`/api/v1/`）

### 五、接口安全设计

1. **认证方式**：JWT / OAuth2 / API Key
2. **限流策略**：每个接口的 QPS 限制
3. **防重放**：请求签名 + 时间戳 + Nonce
4. **敏感数据**：请求/响应中的敏感字段脱敏规则
