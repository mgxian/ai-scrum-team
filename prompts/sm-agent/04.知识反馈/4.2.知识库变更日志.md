## 知识库变更日志

核心指令：记录每次 Sprint 回顾后对各角色知识库/规则的变更，支持追溯和回滚。

---

### 变更日志格式

每次变更按以下格式追加记录：

```
---
变更批次：Sprint {N} 回顾
变更日期：{date}
审批人：{PO_name}
---
```

| 序号 | 条目编号 | 目标角色 | 变更类型 | 变更文件 | 变更前内容 | 变更后内容 | 变更原因 |
|:--|:--|:--|:--|:--|:--|:--|:--|

变更类型：
- ADD：新增规则/知识条目
- MODIFY：修改现有规则/阈值
- DELETE：移除不再适用的规则
- ROLLBACK：回滚到之前版本

### 回滚机制

当某条改进项在后续 Sprint 中被证明不合理（误报增多、效率下降等）：

1. SM Agent 在回顾中标记该条目为"待回滚"
2. PO 确认回滚
3. SM Agent 恢复变更前内容，在日志中记录 ROLLBACK
4. 下个 Sprint 验证回滚效果

### 知识库健康度指标

每 3 个 Sprint 进行一次知识库健康度检查：

| 指标 | 计算方式 | 健康阈值 |
|:--|:--|:--|
| 规则生效率 | 已生效条目 / 已写入条目 | ≥ 70% |
| 规则回滚率 | 已回滚条目 / 已写入条目 | ≤ 15% |
| 重复问题消除率 | 已消除的重复问题 / 总重复问题 | ≥ 60% |
| 知识条目增长率 | 本周期新增条目 / 上周期总条目 | 10%-30%（过快说明基础规则不足，过慢说明回顾不充分） |
