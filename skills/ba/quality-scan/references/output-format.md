# 输出格式

## 质量扫描报告

| 问题编号 | 问题类型 | 问题位置 | 严重程度 | 改进建议 |
|---------|---------|---------|---------|---------|

### 严重程度定义

- **高**：可能导致系统无法使用、重大逻辑错误或安全风险
- **中**：影响部分功能正常使用或用户体验，有明确改进必要
- **低**：表述不规范、冗余或微小遗漏，建议优化

### 问题类型

模糊表达类 / 需求遗漏类 / 冗余处理类 / 表述格式类 / 逻辑错误类 / 工具使用错误类 / 日志缺失类 / 易遗漏处理类 / 重复描述类

### 规则

- 问题位置具体到章节或功能描述（如"第四章摘除 Pod 需求描述"）
- 按严重程度从高到低排序
- 每个问题必须有具体可执行的改进建议
